# 安全测试指南

## 安全测试执行步骤

### 1. 准备测试环境

```bash
# 构建项目
go build -o bin/relay.exe ./cmd/relay
go build -o bin/remote.exe ./cmd/remote

# 生成测试证书
bin/relay.exe --gen-certs --certs-dir ./test-certs
```

### 2. 功能安全测试

#### 测试 S01: 协助码随机性

**目的**: 验证协助码具有足够的随机性，防止猜测攻击

**测试步骤**:
```bash
# 编写脚本生成 10,000 个协助码并检查重复
# 使用单元测试: go test -v ./internal/relay -run TestCodeUniqueness
```

**预期结果**:
- 10,000 个码无重复
- 字符分布均匀

---

#### 测试 S10: TLS 版本强制执行

**目的**: 验证仅接受 TLS 1.3 连接

**测试步骤**:
```bash
# 使用 openssl 尝试 TLS 1.2 连接
openssl s_client -connect localhost:8443 -tls1_2

# 使用 nmap 扫描
nmap --script ssl-enum-ciphers -p 8443 localhost
```

**预期结果**:
- TLS 1.2 连接被拒绝
- 仅显示 TLS 1.3 密码套件

---

#### 测试 S11: 强加密套件验证

**目的**: 验证仅使用安全的加密算法

**检查点**:
- 确认使用 AES-256-GCM 或 ChaCha20-Poly1305
- 确认未使用 3DES, SHA-1 等弱算法

**验证方式**:
```bash
# 使用 SSL Labs 测试（公网服务器）
# 或使用 testssl.sh
```

---

#### 测试 S20-S24: 审计日志验证

**目的**: 验证所有关键操作都被记录

**测试步骤**:
1. 启动服务器并指定审计日志
   ```bash
   bin/relay.exe --listen :8443 --audit ./test-audit.log
   ```

2. 执行完整流程
   - Share 连接
   - Help 使用有效码连接
   - Help 使用无效码连接
   - 正常断开连接

3. 检查日志
   ```bash
   type test-audit.log
   ```

**预期结果**:
- 连接成功事件
- 协助码生成事件
- 会话建立事件
- 失败尝试事件
- 会话关闭事件

---

### 3. 输入验证和模糊测试

#### 测试 S30: 大消息处理

**目的**: 验证大消息不会导致内存溢出或崩溃

**测试方式**:
- 修改测试代码发送 100MB 消息
- 观察服务器内存使用
- 确认服务器仍能正常处理新连接

---

#### 测试 S31: 畸形 JSON 处理

**目的**: 验证无效输入不会导致崩溃

**测试步骤**:
```
发送以下内容:
- 不完整的 JSON: `{"type": "register`
- 无效 UTF-8
- 深度嵌套的 JSON
```

**预期结果**:
- 服务器优雅处理错误
- 不崩溃，继续服务
- 记录错误日志

---

### 4. 边界测试执行

#### 测试 B01-B08: 协助码边界

| 测试 | 输入 | 命令 |
|------|------|------|
| 空码 | `""` | `remote help --code ""` |
| 过短 | `"ABC"` | `remote help --code ABC` |
| 含分隔符 | `"ABCD-EFGHIJ"` | `remote help --code ABCD-EFGHIJ` |
| 含空格 | `"ABCD EFGHIJ"` | `remote help --code "ABCD EFGHIJ"` |
| 大小写混合 | `"aBcDeFgHiJ"` | `remote help --code aBcDeFgHiJ` |

---

### 5. 网络异常测试

#### 测试 E01: 服务器未启动

**命令**:
```bash
bin/remote.exe share --server localhost:9999 --insecure
```

**预期**: 友好的错误提示，不崩溃

---

#### 测试 E02: 连接中途断开

**步骤**:
1. 建立完整连接
2. 终止服务器进程
3. 观察客户端行为

**预期**:
- 客户端检测到断开
- 优雅退出
- 无 panic

---

#### 测试 E10: SSH 服务未启动

**命令**:
```bash
bin/remote.exe share --server localhost:8443 --ssh 127.0.0.1:9999 --insecure
```

**预期**:
- 清晰的错误提示
- 告知无法连接 SSH

---

### 6. TLS/证书测试

#### 测试 E20-E23: 证书验证

| 测试 | 操作 | 预期 |
|------|------|------|
| 自签名证书 | 不使用 --insecure | 连接失败 |
| 自签名 + --insecure | 使用 --insecure | 连接成功 |
| 域名不匹配 | 证书域名与实际不符 | 连接失败（除非 --insecure）|

---
